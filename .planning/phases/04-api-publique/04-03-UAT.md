## Vérif 4 — Intégration curl end-to-end (API-01, API-02)

```bash
export KORVIA_KEY="krv_<votre-clé-de-l-étape-3>"

# A) Liste → 200 + { data, total, page, limit }
curl -s -w "\nHTTP %{http_code}\n" \
  -H "Authorization: Bearer $KORVIA_KEY" \
  http://localhost:3000/api/v1/work-orders | jq .

# B) Sans clé → 401
curl -s -w "\nHTTP %{http_code}\n" \
  http://localhost:3000/api/v1/work-orders

# C) Création → 201 + objet BT
curl -s -w "\nHTTP %{http_code}\n" -X POST \
  -H "Authorization: Bearer $KORVIA_KEY" \
  -H "Content-Type: application/json" \
  -d '{"title":"Test API integration","type":"corrective","priority":"medium"}' \
  http://localhost:3000/api/v1/work-orders | jq .

# D) Corps invalide → 400
curl -s -w "\nHTTP %{http_code}\n" -X POST \
  -H "Authorization: Bearer $KORVIA_KEY" \
  -H "Content-Type: application/json" \
  -d '{"title":"ab","type":"corrective","priority":"medium"}' \
  http://localhost:3000/api/v1/work-orders
```

- [ ] A → HTTP 200, body contient `data`, `total`, `page`, `limit`
- [ ] B → HTTP 401
- [ ] C → HTTP 201, objet BT avec `id`, `number`, `title`
- [ ] D → HTTP 400, body contient `issues`
