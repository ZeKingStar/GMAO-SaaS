| Threat | Severity | Mitigation |
|--------|----------|------------|
| Dashboard leaks subscription financial data — renewal date / plan visible to non-authorized users | Low | Dashboard is already protected by Clerk auth + requireOrgAccess() pattern in layout. Only org members see the dashboard. Renewal date is not sensitive financial data. |
| Billing portal session created for wrong org — CSRF or confused deputy on createBillingPortalSession | High | createBillingPortalSession() reads the authenticated user's org from Clerk session server-side — it does not accept user-supplied org IDs. Stripe portal links are scoped to a customer. |
| Widget shows stale plan after Stripe webhook updates — user sees wrong tier | Medium | Dashboard is a Server Component; plan is re-fetched from DB on every render. No client-side subscription state. Stripe webhook updates DB; next dashboard load reflects change. |
| Null org on dashboard — org.subscription access crashes if org itself is null | Medium | Existing `if (!org) redirect('/onboarding')` guard at line 40 prevents this. `subscription` field added to same query — if org is found, subscription is included (may be null, handled explicitly). |
