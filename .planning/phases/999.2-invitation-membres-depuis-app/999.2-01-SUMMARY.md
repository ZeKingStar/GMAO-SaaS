## What Was Built

### src/lib/invitation-roles.ts
Utilitaire pur (pas de `'use server'`) — importable côté client et serveur :
- `ROLE_ORDER`: `['admin','manager','technician','requester','viewer']`
- `getAvailableRoles(currentRole)`: retourne le sous-tableau à partir du rang courant (D-06)
- `canInviteWithRole(currentRole, target)`: boolean — false si target a un rang supérieur (D-06)
- `toClerkRole(role)`: retourne `` `org:${role}` `` — voir note Assumption A1 ci-dessous

### src/actions/settings.ts (extensions)
Trois nouvelles server actions exportées :
- **`inviteMember(emailAddress, role)`** : guard admin/manager → validation email regex → D-06 check → `clerkClient().organizations.createOrganizationInvitation()`
- **`revokeInvitation(invitationId)`** : guard admin/manager → `revokeOrganizationInvitation()`
- **`listPendingInvitations()`** : guard admin/manager → `getOrganizationInvitationList({ status: ['pending'] })` → retourne `.data`

Refactorisation interne : `getAdminContext()` créé (retourne `{ organizationId, orgId, userId, role }`) ; `getAdminOrg()` préservé comme wrapper `return (await getAdminContext()).organizationId` — zéro régression sur `updateMemberRole`, `updateClosureRequirements`, `updateMemberHourlyRate`, `updateEscalationConfig`.

## Security Notes (Threat Model Coverage)

| Threat | Mitigation implémentée |
|--------|------------------------|
| T-999.2-01 — Elevation of Privilege | `canInviteWithRole(currentRole, role)` enforcé AVANT `createOrganizationInvitation` ; currentRole lu depuis DB via `getAdminContext()`, jamais du client |
| T-999.2-02 — Tampering / Injection | `emailAddress.trim().toLowerCase()` + regex `EMAIL_RE` avant appel Clerk |
| T-999.2-03 — Broken Access Control | `getAdminContext()` en tête des 3 actions — throw 'Accès refusé' pour tout rôle < manager |
| T-999.2-04 — IDOR | `organizationId` provient de `auth().orgId` (Clerk) — Clerk refuse une invitation hors de cet org |
| T-999.2-05 — Spoofing | `orgId` provient exclusivement de `auth()` — UUID Prisma (`organizationId` interne) jamais envoyé à Clerk |

## Assumption A1 — Format de rôle Clerk (VÉRIFICATION REQUISE EN DEV)

Le helper `toClerkRole()` retourne `org:${role}` (ex: `org:technician`).

**Vérification à faire manuellement :** Appeler `clerkClient().organizations.getOrganizationMembershipList({ organizationId: orgId })` et inspecter le champ `role` des membres existants. Si la valeur est `technician` (sans préfixe), changer `toClerkRole` en :
```typescript
export function toClerkRole(role: MemberRole): string {
  return role  // pas de préfixe
}
```
Si la valeur est `org:technician` (avec préfixe), aucune modification nécessaire.
