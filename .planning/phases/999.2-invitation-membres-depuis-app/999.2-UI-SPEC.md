## Layout Contract

### "Inviter un membre" button placement (D-02)

- Location: section header of the Équipe section on `/parametres/organisation`
- Layout: `flex items-center justify-between` row — title "Membres de l'équipe" on the left, button on the right
- Visibility: rendered only when `canManage === true` (same pattern as team-table.tsx line 46)
- Button variant: `default` (primary, ocre filled), size `sm` (h-8 compact)
- Icon: `UserPlus` from lucide-react, 16px, placed before label text

### InviteDialog (D-01)

- Width: `sm:max-w-sm` (384px max), full-width on mobile — matches asset-form-dialog.tsx
- Structure:
  ```
  DialogHeader
    DialogTitle: "Inviter un membre"
  DialogContent (vertical stack, gap-4)
    Field: Label "Adresse e-mail" + Input type="email" placeholder="prenom.nom@exemple.com"
    Field: Label "Rôle" + Select (options filtered by D-06)
  DialogFooter
    Button variant="outline": "Annuler l'invitation" (calls setOpen(false))
    Button variant="default" type="submit" disabled={pending}: "Envoyer l'invitation"
  ```
- Default role pre-selected: `technician` (D-07)
- Loading state: submit button shows `disabled` + cursor-not-allowed while `pending === true` (useTransition pattern)
- On success: dialog closes, sonner `toast.success(...)` fires
- On error: dialog stays open, sonner `toast.error(...)` fires

### PendingInvitationsSection (D-03, D-04, D-05)

- Visibility: component renders nothing (`return null`) when `pendingInvitations.length === 0` (D-05)
- When visible: rendered below TeamTable, separated by `mt-8` (32px)
- Section title: "Invitations en attente" — `text-base font-medium` (16px/500), `mb-4` (16px below title)
- Row structure per invitation:
  ```
  div.flex.items-center.justify-between.py-2.border-b.border-border
    div.flex.items-center.gap-2
      span: email address (text-sm font-normal)
      Badge: role label from ROLE_LABELS, variant from ROLE_VARIANTS
    div.flex.items-center.gap-4
      span: "Envoyé le DD MMM YYYY" (text-xs text-muted-foreground)
      Button variant="ghost" size="sm": "Révoquer l'invitation" (text-destructive hover:text-destructive)
  ```
- Date format: `new Intl.DateTimeFormat('fr-CA', { day: '2-digit', month: 'short', year: 'numeric' }).format(new Date(invitation.createdAt))` — produces "30 mai 2026"
- Last row: no bottom border (`last:border-b-0`)

### Destructive Actions

**Révocation d'invitation**

- Action: clicking "Révoquer l'invitation" on a pending invitation row
- Pattern: **inline toast + undo** — do NOT use a confirmation dialog
  - Rationale: Révocation is low-stakes (the person was only invited, not removed from the org). A confirmation dialog would add friction for a routine admin action. Toast + undo matches the pattern used for role changes in team-table.tsx.
  - Implementation: `toast("Invitation révoquée", { action: { label: "Annuler", onClick: () => { /* re-invite not feasible — show info */ } } })` — since Clerk revocation is irreversible, the "Annuler" action in the toast should instead show `toast.info("La révocation ne peut pas être annulée. Vous pouvez renvoyer une invitation.")`.
  - Alternative (simpler): omit undo entirely — just `toast.success("Invitation révoquée")`. Preferred for simplicity given Clerk's irreversible revocation.

**Recommended approach:** Simple `toast.success("Invitation révoquée")` on success, `toast.error("Erreur lors de la révocation")` on failure. No confirmation dialog, no undo.
