### Human Verification Required

#### 1. Format du rôle Clerk — Assumption A1

**Test:** Se connecter à l'app en dev avec un compte admin. Depuis la console ou un route de debug, appeler `clerkClient().organizations.getOrganizationMembershipList({ organizationId: orgId })` et inspecter le champ `role` des membres existants.

**Expected (si préfixe correct):** Le champ `role` retourne des valeurs comme `org:admin`, `org:manager`, `org:technician`. Aucune modification nécessaire dans `toClerkRole()`.

**Expected (si pas de préfixe):** Le champ `role` retourne des valeurs comme `admin`, `manager`, `technician`. Modifier `toClerkRole()` dans `src/lib/invitation-roles.ts` :
```typescript
export function toClerkRole(role: MemberRole): string {
  return role  // pas de préfixe
}
```

**Why human:** La valeur correcte du format de rôle Clerk dépend de la configuration de l'instance Clerk Korvia spécifique. Il est impossible de déterminer cela statiquement depuis le code. C'est documenté comme "Assumption A1" dans le SUMMARY Plan 01 avec instruction de vérification manuelle en dev.
